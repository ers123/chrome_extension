제품 요구사항서 (PRD) — Tab Nudge

Product Overview
	•	비전: 브라우저 탭 과잉(탭 폭주)으로 인한 인지 부하와 성능 저하를 줄여, 사용자가 더 적은 탭으로 더 집중해서 일할 수 있게 돕는다.
	•	미션: “지각(Awareness) → 제안(Suggestion) → 행동(Action)”의 흐름으로, 사용자가 과도한 탭 상태를 즉시 인지하고 한 번의 클릭으로 정리 행동을 취하도록 넛지한다.
	•	요약: 사용자가 설정한 임계치 이상으로 탭이 증가하면 알림을 띄우고, 알림 내에서 오래된 탭 닫기, 중복 탭 닫기, 도메인별 묶기/보관, 일시중지(스누즈) 등의 즉시 실행 액션을 제공한다. 선택적으로 주간 리포트, 점수/배지, 스마트 제안(주제별 그룹화·비활성 탭 절전·읽기목록 제안) 및 대시보드를 제공한다.

Objectives & Goals
	•	비즈니스 목표
	•	3개월 내 크롬 웹스토어 공개 및 초기 5,000 DAU 확보
	•	8주차 유료 전환(프리미엄 기능) A/B 테스트, 전환율 3% 달성
	•	사용자 목표
	•	활성 사용자 기준 평균 동시 열림 탭 수(Avg Concurrent Tabs) 20% 감소
	•	알림 표시 후 1분 이내 정리 액션 실행율 35% 이상
	•	성공지표(상위)
	•	D7/D30 유지율 ≥ 35%/20%
	•	알림 무시율 ≤ 40%
	•	주당 평균 정리 액션 수 ≥ 4회/사용자

Target Audience
	•	페르소나
	•	파워 유저 민수(29, 기획자): 리서치 중 탭 50+ 유지. 정리는 미루고 집중이 자주 깨짐.
	•	개발자 지현(33, 프론트엔드): 문서/이슈/PR/빌드 탭 중복 다수. PC 메모리 압박 체감.
	•	학생 아라(22, 대학생): 과제·참고자료 탭을 “나중에 보기”로 쌓아두고 실제로는 잘 안봄.
	•	주요 사용 시나리오
	•	리서치/학습/개발 등 다중 문맥 전환이 많은 업무
	•	장시간 브라우징으로 탭 과잉과 브라우저 성능 저하 경험

User Stories & Use Cases
	•	US-01: “탭이 너무 많아졌을 때, 바로 어떤 탭을 정리할지 추천받고 한 번에 처리하고 싶다.”
	•	US-02: “같은 페이지를 여러 개 띄워둔 중복 탭을 자동으로 찾아 닫고 싶다.”
	•	US-03: “지금은 집중해야 하니, 일정 시간 알림을 쉬고 싶다.”
	•	US-04: “일주일 동안 내 탭 습관을 숫자로 보고 개선을 체감하고 싶다.”
	•	US-05: “오랫동안 보지 않은 탭은 자동으로 절전하거나 읽기목록으로 보내고 싶다.”
	•	US-06: “설치 직후 간단히 임계치만 정하고 바로 쓰고 싶다.”

Functional Requirements

각 항목은 고유 ID와 수용 기준(AC)을 포함합니다.

FR-001 임계치 기반 알림
	•	설명: 사용자가 설정한 탭 개수 임계치 도달/초과 시, 즉시 알림 제공.
	•	세부요건
	•	기본 임계치 30, 범위 5–200, 1단위 조정
	•	재알림 쿨다운(기본 10분) 설정 가능
	•	AC
	•	탭 수가 임계치를 초과하면 2초 이내 알림 표시
	•	알림에는 현재 탭 수와 임계치 표시
	•	쿨다운 시간 내에는 동일 조건에서 재알림 미표시
	•	설정에서 임계치/쿨다운 변경 후 즉시 반영

FR-002 액션 가능한 알림 버튼
	•	설명: 알림 내 즉시 실행 액션 제공.
	•	액션
	•	A1: 가장 오래된 탭 N개 닫기 (기본 10, 5–30 선택)
	•	A2: 중복 탭 닫기 (정규화된 URL 기준)
	•	A3: 특정 도메인 묶기/보관 (선택한 도메인 탭을 묶어 보관함으로 이동)
	•	A4: 1시간 스누즈 (15/30/60/180분 옵션)
	•	AC
	•	각 액션 클릭 시 1초 이내 처리 시작 피드백 제공
	•	실행 전 요약(닫힐/이동될 탭 수) 표시 및 확인 가능
	•	실행 후 탭 수가 기대치와 일치
	•	10초간 되돌리기(Undo) 제공, 실행 전 상태로 완전 복구

FR-003 대시보드 & 통계
	•	설명: 현재·과거 탭 지표와 추이 가시화.
	•	지표: 평균/최대 동시 탭 수, 일일 신규 탭 수, 중복 비율, 상위 도메인, 액션 사용량
	•	AC
	•	대시보드 페이지 로드 ≤ 2초(캐시 기준)
	•	최근 7/30일 전환 토글 제공
	•	CSV 내보내기 제공(UTF-8, 헤더 포함)

FR-004 주간 리포트 & 점수/배지(옵션)
	•	설명: 주 1회 요약 리포트와 관리 점수, 도전과제·배지 제공(사용자 옵트인).
	•	AC
	•	매주 일요일 09:00(로컬) 발송 또는 표시
	•	지난주 대비 변화(증가/감소 %) 표시
	•	최소 3종 배지(초보/꾸준/마스터) 부여 로직 적용
	•	설정에서 온/오프 및 알림 채널 선택 가능

FR-005 스마트 제안(옵션, 고급)
	•	설명: 사용 패턴 기반 제안.
	•	S1: 주제별 그룹화 제안 (유사 제목/도메인/메타 기반)
	•	S2: 비활성 탭 절전 제안 (X분 미사용 시)
	•	S3: 읽기목록 이동 제안 (Y일 미열람 기사성 탭)
	•	AC
	•	각 제안은 근거(예: “Notion 관련 5개”)와 예상 효과(“탭 5↓”) 표시
	•	사용자는 제안별 항상 허용/한시 허용/거부 선택 가능
	•	제안 빈도는 사용자 피드백에 따라 자동 조절(거부 ↑ ⇒ 빈도 ↓)

FR-006 온보딩 & 설정
	•	설명: 30초 내 시작 가능한 간결 온보딩과 설정 화면.
	•	AC
	•	온보딩 3단계 이내(임계치, 스누즈 기본값, 리포트 옵트인)
	•	모든 주요 기능 토글/임계치/시간값 편집 가능
	•	설정 검색 제공(예: “중복” 입력 시 관련 항목 하이라이트)

FR-007 접근성 & 단축키
	•	설명: 접근성 표준과 단축키 제공.
	•	AC
	•	키보드만으로 알림 내 모든 액션 실행 가능(Tab/Enter)
	•	스크린리더로 알림·버튼 라벨 정확 낭독
	•	기본 단축키 제공(예: 알림 열기, A1~A4 실행), 사용자 재지정 가능

FR-008 국제화
	•	설명: 한국어/영어 제공, 추가 언어 확장 가능.
	•	AC
	•	언어 전환 시 UI 텍스트 100% 반영, 재시작 불필요
	•	날짜/숫자/시간 로케일 형식 준수

Non-Functional Requirements
	•	성능: 백그라운드 평균 CPU 점유 ≤ 2%, 메모리 오버헤드 ≤ 100MB
	•	지연: 알림 표시 ≤ 2초, 액션 피드백 시작 ≤ 1초
	•	신뢰성: Undo 복구 성공률 ≥ 99.9%, 치명적 오류 발생률 ≤ 0.1%/주
	•	호환성: 최신 크로미움 계열 브라우저 지원, 다중 윈도우/프로필 대응
	•	프라이버시: URL/콘텐츠는 기본 로컬 처리, 텔레메트리는 옵트인·익명화·도메인 수준 집계
	•	보안: 최소 권한 원칙, 민감 데이터 저장 암호화, 외부 통신 도메인 화이트리스트
	•	사용성: 주요 정리 액션까지 2클릭 이내, 온보딩 30초 이내 완료
	•	가용성: 알림/액션 기능 가동시간 월 ≥ 99.5%

Success Metrics (KPIs)
	•	행동 전환: 알림→액션 클릭률 ≥ 35%
	•	효율 개선: 사용자별 피크 탭 수 중앙값 20% 감소(설치 후 14일 내)
	•	참여도: 주간 리포트 열람률 ≥ 30%, 배지 달성 사용자 비중 ≥ 25%
	•	만족도: NPS ≥ 30, 지원 티켓/1000DAU ≤ 5
	•	유지/수익: D30 ≥ 20%, 프리미엄 전환율 ≥ 3%

Risks & Assumptions
	•	권한 제약/스토어 정책: 과도 권한 사용 시 검수 지연 가능 → 최소 권한/투명 고지/옵트인 원칙
	•	성능 리스크: 탭 스캔/중복 탐지로 리소스 증가 가능 → 샘플링/빈도 제한/아이들 타임 활용
	•	프라이버시 우려: URL 처리 민감 → 로컬 처리 기본, 익명 도메인 통계만 수집
	•	행동 피로: 알림 과다로 무시/제거 → 빈도 제한, 개인화, 학습 기반 노이즈 감소
	•	사용자 습관 고착: 읽기목록 적체 → 리마인더·만료 정책·재표면화 카드 제공
